Pipes:
1) Stream pipes: безструктурная отправка информации. Всегда однонаправлены
2) Message pipes: структурная отправка информации. Двунаправлены, однако данные могут отправляться в одном направлении

Default Control Pipe:
Позволяет идентифицировать устройство. Имеет:
1) Standard information для всех USB устройств.
2) Class information зависит от класса USB устройства, как идентифицируется с помощью стандартной информации
3) USB Vendor information свободна для использования производителем устройства

Основы USB передачи:
Пакеты:
1) A token packet показывает тип и направление передачи, адрес устройства и номер endpoint
2) В зависимости от направления передачи, host или function (устройство) посылает data packet (который может просто показывать, что данных нет)
3) Получающее устройство отвечает с помощью handshake packet чтобы показать, что передача была успешной

4 основных типа передачи данных:
1) Control Transfers предоставляют передачи без потери (lossless transmission) и используются для того, чтобы сконфигурировать устройство. Таким образом все USB устройства должны поддерживать control transfers хотя бы с помощью default control pipe
2) Bulk Data Transfers предоставляют последовательную передачу без потерь и обычно используются для передачи большого количества информации
3) Interrupt Data Transfers предоставляют надежную, с ограниченной задержкой, передачу данных. Обычно используется в human input deviced, таких как мышь или джойстик
4) Isochronous Data Transfers, иначе называемые Streaming Real-time Transfers, устанавливают необходимую пропускную способность и задержку когда инициализируют передачу данных. Этот тип передачи данных преимущественно используется в таких приложениях, как streaming audio. Потому что частота отправки данных более важна, чем целостность данных, для этого типа передачи данных, он не предоставляет никакой механизм проверки ошибок и исправления
